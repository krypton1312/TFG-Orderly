  </files>
  <action>
**For each Android app (Phone and Tablet), do:**

1. Remove `android:usesCleartextTraffic="true"` from `src/main/AndroidManifest.xml` `<application>` tag.

2. Create `src/debug/AndroidManifest.xml` (new file) with ONLY the cleartext override:
   ```xml
   <?xml version="1.0" encoding="utf-8"?>
   <manifest xmlns:android="http://schemas.android.com/apk/res/android">
       <!--
           Allows cleartext HTTP for local LAN connections during development and demo.
           This file is only merged into DEBUG builds. Release builds use HTTPS only.
           Required for: local Wi-Fi bar network (192.168.x.x) and emulator (10.0.2.2).
       -->
       <application android:usesCleartextTraffic="true" />
   </manifest>
   ```

The Android manifest merger automatically merges `src/debug/AndroidManifest.xml` into debug builds only. No changes to build.gradle needed.
  </action>
  <verify>
    Confirm main manifest no longer has usesCleartextTraffic:
    - grep "usesCleartextTraffic" in src/main/AndroidManifest.xml → no results
    - grep "usesCleartextTraffic" in src/debug/AndroidManifest.xml → 1 result
    Both apps must still assemble debug without error: `.\gradlew.bat assembleDebug`
  </verify>
  <done>
    - src/main/AndroidManifest.xml has no usesCleartextTraffic attribute
    - src/debug/AndroidManifest.xml exists with the cleartext override
    - Debug APK builds successfully for both apps
  </done>
</task>
