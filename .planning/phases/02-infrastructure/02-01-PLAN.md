<objective>
Make the phone app's backend URL configurable without changing source code. SERVER_HOST is set in local.properties (git-ignored) and exposed via BuildConfig. Changing the IP for a real device demo requires only updating local.properties and rebuilding — no source edit.

Purpose: INF-01 — phone must reach the backend on real Android devices, not just the emulator alias 10.0.2.2.
Output: AppModule.kt reads BASE_URL from BuildConfig, local.properties holds the server IP.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Expose SERVER_HOST via BuildConfig</name>
  <files>frontend/OrderlyPhone/local.properties, frontend/OrderlyPhone/app/build.gradle.kts</files>
  <action>
1. In `local.properties` (phone root), add:
   ```
   SERVER_HOST=10.0.2.2
   ```
   (keeps emulator default; real device → change to LAN IP before build)

2. In `app/build.gradle.kts`, add at top after existing imports:
   ```kotlin
   import java.util.Properties
   val localProps = Properties().apply {
       rootProject.file("local.properties").takeIf { it.exists() }
           ?.inputStream()?.use { load(it) }
   }
   ```

3. Inside `android { }` block, add:
   ```kotlin
   buildFeatures {
       buildConfig = true
   }
   ```

4. Inside both `debug { }` and `release { }` buildTypes (or at defaultConfig level):
   ```kotlin
   buildConfigField("String", "SERVER_HOST", "\"${localProps.getProperty("SERVER_HOST", "10.0.2.2")}\"")
   ```
  </action>
  <verify>./gradlew generateDebugBuildConfig — produces BuildConfig.java with SERVER_HOST field. No compile error.</verify>
  <done>BuildConfig.SERVER_HOST exists and equals the value in local.properties</done>
</task>

<task type="auto">
  <name>Task 2: Wire AppModule to BuildConfig</name>
  <files>frontend/OrderlyPhone/app/src/main/java/com/example/orderlyphone/di/AppModule.kt</files>
  <action>
Replace the hardcoded constant:
```kotlin
// BEFORE:
private const val BASE_URL = "http://10.0.2.2:8080/" // эмулятор

// AFTER:
private val BASE_URL get() = "http://${BuildConfig.SERVER_HOST}:8080/"
```
Import: `import com.example.orderlyphone.BuildConfig` (may already be auto-imported).

The `val` with `get()` ensures it reads BuildConfig at call time (not class init), which is correct since BuildConfig is a compile-time constant anyway.
  </action>
  <verify>./gradlew assembleDebug — APK builds without errors. BASE_URL in AppModule no longer contains a hardcoded IP string literal.</verify>
  <done>AppModule uses BuildConfig.SERVER_HOST; grep for "10.0.2.2" in AppModule.kt returns no results</done>
</task>

</tasks>
